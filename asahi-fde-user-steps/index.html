디스크 암호화: 사람이 하는 일은 세 줄뿐

이미 쓰고 있는 M1 Max 맥북(Asahi Linux)의 디스크를 나중에 통째로 암호화(LUKS)하는 작업에서, 사람이 직접 하는 일만 추린 안내판이다. 기술 명령 대부분은 자동화 에이전트가 원격으로 실행한다. 사람 몫은 네 가지: 시작 전 백업, 재부팅 뒤 로그인, 검은 셸에서 세 줄 입력, 그리고 이후 부팅마다 암호 입력.

2026-07-15 · 개인 작업 참조용(목록 비공개)

0. 시작 전 (콘솔에 앉기 전에 끝내둘 것)

암호화는 되돌리기 번거로운 작업이라, 앉기 전에 아래를 먼저 갖춘다.

  • 설정·데이터 백업. 시스템 설정 포크는 원격 저장소로, /home은 별도 저장소(NAS 등)로 미리 넘겨둔다. 모델·비디오·빌드 캐시처럼 다시 받으면 되는 큰 파일은 빼면 시간이 크게 준다.
  • 새 디스크 암호 정하기. 영문·숫자만(ASCII). 앞으로 부팅할 때마다 입력하게 된다.
  • AC 전원 + 1~2시간 확보. 변환 도중 전원이 끊기면 안 된다.

백업이 끝나면 바로 착수할 수 있다.

1. 전체 그림: 누가 뭘 하나

단계에이전트가 함 (원격)사람이 함 (콘솔)
1. /boot 이동명령 전부 + 재부팅재부팅 지켜보고 로그인
2. 훅 추가·FS 축소명령 전부없음
3. 변환 부팅부팅 옵션 설정 + 재부팅검은 셸에서 세 줄 입력 ← 핵심
4. 암호 부팅 확정설정 정리 + 재부팅암호 입력하고 로그인
5. 검증확인 스크립트콜드부팅·재부팅·서스펜드 각 1회

재부팅 때마다 에이전트 연결이 끊긴다. 로그인이 되면 에이전트가 다시 붙어 다음 단계를 진행한다.

2. 사람 차례 ①: 재부팅 확인 (1단계 끝)

에이전트가 재부팅을 걸면 화면이 꺼졌다 켜진다. 평소처럼 로그인 화면이 뜨면 로그인하면 끝. 이 시점은 아직 아무것도 암호화하지 않았으므로, 로그인 화면이 안 떠도 데이터는 100% 안전하다(에이전트가 준 복구 절차로 되돌린다).

3. 사람 차례 ②: 검은 셸에서 세 줄 (3단계, 유일한 직접 입력)

어느 재부팅에서 로그인 화면 대신 검은 텍스트 화면이 나오고 [rootfs ]# 프롬프트에 멈춘다. 고장이 아니라 일부러 만든 정지다. 이 순간엔 네트워크가 없어서 에이전트가 못 들어온다. 아래 세 줄을 순서대로 입력한다(시작 전에 인쇄용 카드로도 준비해두면 좋다).

cryptsetup reencrypt --encrypt --type luks2 --reduce-device-size 32M /dev/nvme0n1p5
  • 새 디스크 암호를 두 번 물어본다. 정해둔 암호를 입력.
  • 진행률(%)이 올라간다. 수십 분 걸린다. 전원 절대 유지. 끝날 때까지 기다린다.
cryptsetup open /dev/nvme0n1p5 root
  • 방금 만든 암호를 다시 한 번 입력.
exit
  • 부팅이 그대로 이어져 로그인 화면까지 간다. 로그인하면 에이전트가 다시 붙는다.
키보드 배열 주의: 이 셸도, 부팅 암호 프롬프트도 평소 로그인과 같은 배열을 쓴다(예: 드보락을 쓰면 여기서도 드보락). 평소 치던 대로 치면 된다.

4. 사람 차례 ③: 암호 부팅 확인 (4~5단계)

4단계 재부팅부터는 부팅 초반에 디스크 암호를 물어본다. 입력하고 로그인까지 되면 성공. 마지막으로 다음을 각 1회 확인한다(에이전트가 순서 안내):

  • 완전히 껐다 켜기(콜드부팅) 후 암호 입력 → 정상 로그인
  • 재부팅 → 정상
  • 뚜껑 닫아 서스펜드 → 열어서 복귀 정상
  • 커널 업데이트 1회 후 재부팅 정상

여기까지 통과해야 "끝". 그 전까지 백업은 지우지 않는다.

5. 문제가 생기면

구간데이터대처
1~2단계 (변환 전)평문 그대로, 100% 안전에이전트가 설정만 되돌리면 원상복구
3단계 변환 중변환 중 (전원만 유지하면 안전)전원 차단만 금지. 중단돼도 재개 가능하나 위험
4단계 이후암호화 완료부팅 실패 시 긴급 셸에서 복구 (에이전트 카드 참조)

당황하지 말 것: 어느 단계든 화면에 보이는 메시지를 사진 찍어두면 에이전트가 판단한다.
